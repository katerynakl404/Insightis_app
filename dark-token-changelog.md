## Перейменування токенів (naming alignment)

Prod вже встановлює ці токени з правильними значеннями, але під іншими іменами. Компоненти що читають старі імена отримують значення з `:root` (light theme). Потрібно або мігрувати компоненти, або додати аліаси.

| Kit (очікуване ім'я) | Prod (поточне ім'я) | Значення в `.dark` |
|----------------------|---------------------|-------------------|
| `--ink` | `--ink-primary` | `var(--grey-50)` ✓ |
| `--card` | `--surface-card` | `var(--grey-900)` ✓ |
| `--card2` | `--surface-card2` | `var(--grey-800)` ✓ |
| `--chips` | `--surface-chips` | `var(--grey-700)` ✓ |
| `--border` | `--stroke-border` | `var(--grey-700)` ✓ |
| `--border-hover` | `--stroke-border-hover` | `var(--slate-700)` ✓ |
| `--focus-ring` | `--state-focus-ring` | `var(--slate-100)` ✓ |
| `--fb-red-hover` | `--fb-error-hover` | `var(--red-800)` ✓ |
| `--fb-red-press` | `--fb-error-press` | `var(--red-850)` ✓ |

| Токен | Потрібне значення | Наслідок відсутності |

| `--btn-secondary-bg` | `var(--card)` (auto-resolves ✓) | Ок — через cascade |

## Пріоритет виправлень

| Пріоритет | Токен | Причина |
|-----------|-------|---------|
| **P1** | `--ink-secondary` / `--text-secondary` | 59% vs 84% — текстова ієрархія ламається |
| **P1** | `--ink-inactive` / `--text-light` | 32% vs 64% — inactive-елементи майже невидимі |
| **P1** | `--state-hover` / `--hover` | teal hover замість нейтрального |
| **P1** | `--card` | фіолетовий cast на всіх card-поверхнях |
| P2 | `--background` / `--bg` | неправильний page-bg |
| P2 | `--border` / `--stroke-border` | фіолетовий cast + naming alignment |
| P2 | Всі naming-аліаси | Компоненти на old-token-API отримують light-theme значення |
| P3 | `--ink-body` / `--text-body` | невелика різниця яскравості |

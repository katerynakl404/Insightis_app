# design-sync notes — Insightis kit

- This repo is NOT a React package: the design system is plain HTML/CSS (`insightis-preview-kit.html` + `pages/kit-theme.css`). The synced package is `ds-react/` — hand-authored thin React wrappers that render the exact kit classes; ALL styling stays in `pages/kit-theme.css` (single source of truth, copied verbatim into `ds-react/dist/kit.css` at build time with the DM Sans Google-Fonts `@import` prepended → expect `[FONT_REMOTE]`, that's correct).
- When kit-theme.css or kit markup changes, re-sync = rebuild `ds-react` (`npm run build` in ds-react/) then run the driver. If a component's kit markup changed, update its wrapper in `ds-react/src/<Name>.tsx` first.
- Node.js was absent on this machine; installed via winget (OpenJS.NodeJS.LTS) on 2026-07-20 with user approval. Playwright chromium install also user-approved.
- Dark theme = `.dark` class on any ancestor (kit re-declares semantic tokens under `.dark`); previews/designs can theme a subtree with `<div className="dark">`.
- `s-hover`/`s-focus`/`s-pressed` classes in kit markup are storybook state-simulation classes — wrappers deliberately do NOT expose them as props; real states only (disabled, error, checked, loading, open...).

## Wrapper-authoring findings (2026-07-20, first sync)

- Kit-internal inconsistencies spotted (kit-side, not fixed by the sync): #spinner section circular demo uses track `var(--chips)` while canonical #circularprogress uses `var(--bg)` (wrappers follow #circularprogress); `components/ProviderCard.html` uses `badge-success` which does not exist in kit-theme.css (real class: `badge-green`); DataSourceCard flame badge offset −5px in storybook vs −4px in the approved connections page (wrapper follows the approved page); tooltip `left` side named in prose but has no CSS rule; Datepicker "today" state marked "needed" in storybook, no CSS yet; `.sep` has no label class.
- Class-name gotchas: DropZone family is `.dsf-drop` (no `.dropzone`); Pagination has NO dedicated class — pure composition of iconbtn/btn with kit-canonical inline 2rem sizing, container supplies the gap; generic card class is `.card-c` (bare `.card` is not a component class); Modal close / File actions / PasswordInput eye use kit-canonical inline sizing styles (no dedicated classes exist).
- `Sidebar` brand logo renders `<use href="#lg">`/`#lg-mark` — pages/previews must define those SVG symbols (they live at the top of insightis-preview-kit.html); previews for Sidebar must inline the symbol defs or the logo area renders empty.
- `MetaRow` per kit = count + inline-actions caption row (not icon+label+value).
- Dark theme CANNOT be demoed on a nested `<div class="dark">`: kit tokens are chains (`--btn-secondary-bg:var(--card)`) resolved at the DECLARING element (:root), so a nested .dark only re-themes tokens explicitly re-declared in the `.dark{}` block. The kit itself toggles `dark` on `<html>`. Rule for previews + the design agent: apply `dark` at the document root only. Preview cards therefore show light theme only.

# Sidebar — prod → expected

Source: `@insightis/ui` `Sidebar/` — overall component plus the sub-parts SidebarHeader, SidebarContent, SidebarFooter, SidebarGroup, SidebarMenu, SidebarMenuButton, SidebarMenuSub, SidebarMenuSubButton, SidebarRail, SidebarTrigger, SidebarInset, SidebarProvider. Baseline: [`../current/Sidebar.md`](../current/Sidebar.md).

**Structural redesign of `SidebarContent` + `SidebarFooter`.** The shell, sub-part API, width tokens, variants and collapse modes are unchanged. What changes is the *information architecture* inside `SidebarContent` (header brand row + CTA → 2 compact nav rows → Pinned/Recent chat sections with persistent gradient-fade labels, hover-revealed kebab menus, and per-row status indicators) and the *density* of `SidebarFooter` (compact tokens meter + 2-line user row).

## A11y notes
- Active nav row: render with `aria-current="page"` (consumer responsibility; documented). Color-only identification is paired with this semantic so the active item is also announced, never color-alone.
- Disabled nav row: `aria-disabled="true"` *in addition* to the opacity treatment.
- Chat row kebab carries `aria-label="More actions"`, `aria-haspopup="menu"`, `aria-expanded` toggled by the consumer. Menu items follow `role="menuitem"` semantics from the existing Dropdown contract.
- Chat row status indicators are paired with `<span class="sbx-sr">In progress / New activity</span>` (visually hidden but read by assistive tech). Row gets `aria-busy="true"` while loading.
- Hit target ≥ 24×24 on every interactive element (nav row 32 expanded / 28×28 collapsed, chat row 28 × full row width, kebab 24×24, section head 24, user row ≥ 28).
- Focus ring: 2px `--focus-ring` over a 2px `Surface/Card` gap — visible against both the default and `State/Hover` surfaces.
- "See all" link is hover-only by default but **also revealed on `:focus-visible`** so keyboard users can reach it; below `1024 px` (and on `hover:none` devices) it is forced `opacity:1` so touch users can reach the chats library without a hover state.
- Scrollbar visually hidden but `overflow:auto` retained — keyboard/wheel scrolling preserved.

## Responsive behaviour (this iteration)

Two-stage responsive. On **desktop** (`≥ 1024 px`) the sidebar lives in the layout column with optional in-place icon-mode collapse. Below `1024 px` (tablet + mobile) the sidebar is **fully hidden** (no icon rail) and surfaces only as an **overlay** opened from a topbar hamburger. Same overlay/backdrop pattern the Metrics page uses for its right panel.

**Breakpoint note:** two distinct thresholds are in play and must not be conflated. The hide-and-overlay-drawer behaviour is gated at **`@media (max-width:1023px)`** (the `.cl-side{display:none}` / overlay rules) — so the sidebar disappears below **1024 px**. Separately, the in-place icon-rail rules (`.sbx.is-collapsed`) are wrapped in **`@media (min-width:768px)`** purely so the `.is-collapsed` class produces no stray layout in the drawer; the `768 px` figure does **not** mark where the sidebar hides.

| Breakpoint | Layout | Width | Open affordance | Behaviour |
|---|---|---|---|---|
| **≥ 1024 px** (desktop) | In-flow fixed-width column | `SIDEBAR_WIDTH` **`16rem`** — page layout (`.cl-shell{grid-template-columns:16rem 1fr}`, `.sbx{width:100%}`); the kit `.sbx` storybook card matches at `16rem`. | In-sidebar collapse trigger (`.sbx-collapse`) | Visible by default. Trigger toggles **in place** to `SIDEBAR_WIDTH_ICON` `3.563rem` (icon-only) and back. No overlay. |
| **< 1024 px** (tablet + mobile) | Hidden by default → overlay on demand | `min(18rem, 86vw)` when open; otherwise not rendered | **Topbar hamburger** (`.cl-burger`, 2rem square, 18px glyph) | `.cl-side{display:none}` until the user taps the hamburger. Tap → sidebar mounts as a **position:fixed overlay** at `min(18rem,86vw)` with a semi-opaque backdrop dimming the whole page. The in-sidebar trigger acts as a **close** button (same as backdrop). |

**Implementation contract** (used by `pages/approved/chat-landing.html`):
- The page topbar carries a `.cl-burger` button — `display:none` on desktop, `display:inline-flex` below `1024 px` (2rem square, radius `.375rem`, color `--ink-body`, hover bg `--state-hover`, focus `--shadow-focus`). Its onclick adds `.side-open` to `<html>` and removes `.is-collapsed` from `.sbx` (ensures the overlay shows in its full state, not desktop's icon mode).
- The in-sidebar trigger (`.sbx-collapse`) is one onclick that branches on viewport: ≥ 1024 px → toggle `.sbx.is-collapsed`; < 1024 px → remove `.side-open` (close overlay).
- **Collapsed icon-rail tooltips.** In the collapsed rail the nav labels are hidden, so each rail item (`.sbx-nav-item`, `.sbx-chats-icon`) shows a **hover tooltip** with its label, using the page's standard JS tooltip engine (300 ms warm-up delay). The engine derives the text from the item's hidden `.lbl` (falling back to `aria-label`) and is gated on `el.closest('.sbx.is-collapsed')` — so tooltips appear **only when collapsed**; the expanded sidebar shows its visible labels with no duplicate tooltip. In the collapsed rail the tooltip is placed **to the right of the icon, vertically centred** (flipping to the left edge if the viewport can't fit it) — not above, since the rail hugs the screen's left edge. Non-rail tooltips keep the default above/below placement.
- A backdrop element (`<div class="cl-side-backdrop">`) is `display:none` on desktop and `display:block` below `1024 px`. It covers the **full viewport** (`position:fixed; inset:0; z-index:55`) including the topbar — background `var(--overlay-scrim)`, with `backdrop-filter:blur(2px)`. Its `opacity` transitions `0 ↔ 1` (`.2s ease`) based on `html.side-open`, and `pointer-events` flips so clicks pass through when closed. Clicking it removes `.side-open`.
- Below `1024 px`, the sidebar is `display:none`. With `html.side-open`, it becomes `display:flex; position:fixed; top:0; left:0; bottom:0; width:min(18rem,86vw); z-index:60` with a soft drop shadow (`0 10px 30px rgba(2,6,23,.32)`) and a `clSideSlide` keyframe slide-in (`translateX(-100%) → 0` over `.22s ease`). While open, `.cl-burger` and the sticky `.cl-page-head` are `visibility:hidden` so they don't pierce the backdrop. Internal layout (header / content / footer) is the kit's normal sbx — no separate mobile component.
- JS resize handler removes `.side-open` whenever the viewport crosses back to ≥ 1024 px, so the sidebar resumes its in-flow layout cleanly.
- Below `< 640 px`, the main column also tightens: padding `.75rem 1rem`, title font-size `1.5rem`, composer padding `.5rem`, hero gap `1.25rem`. The composer footer rows allow `flex-wrap:wrap` so the Send button can drop to its own row instead of forcing label wrap mid-word.
- Below `< 480 px`, dropdown labels (`.cl-dd-lbl`) and the Send button label (`.cl-send-lbl`) collapse to **icon-only**; SVG glyphs + `aria-label` retain meaning. Suggestion pills tighten (h32, 12px font).

The previous Sheet-based mobile rendering (kit baseline — `Sheet side="top"`) is **superseded** by this pattern. The icon-mode rail is intentionally **not** rendered on mobile: at narrow widths every pixel of main-column belongs to content, not navigation chrome.

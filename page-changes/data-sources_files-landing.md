## Hard rules — read before editing this page

These are locked decisions. Do not change without explicit approval.

| # | Rule |
|---|---|
| 1 | Drop zone is always visible above the file list regardless of file count. Never hide or collapse it when files exist. |
| 2 | File row kebab (···) is hidden at rest (`opacity:0`) and fades in on row hover — same pattern as Chats Library. |
| 3 | File row checkbox is hidden at rest and appears on row hover or when any item is selected — same pattern as Chats Library. |
| 4 | The whole **library block** (`.dsf-library` = meta row + filter chips + list) is hidden when file count is 0 — only the drop zone and the first-run empty state (rule 26) remain. |
| 5 | Primary action button stays in the title row. Do not move it into the tab bar. |
| 6 | Drop zone drag state is controlled by the `.is-dragover` class (adds `background:var(--state-hover)` + `border-style:solid`). Do not use inline JS styles for drag state. |
| 7 | Drop zone is a **kit component** (`.dsf-drop*` in `kit-theme.css`, storybook `#dropzone`, [changes/DropZone.md](../changes/DropZone.md)) — a centered dashed drop target with the upload icon **inline with the title**, helper line + **Browse Files** button. It has full states: **rest / hover / focus (`:focus-within`) / drag-over (`.is-dragover`)**. Do not make it a horizontal bar, keep the icon on the title's row, and don't re-add page-level `.dsf-drop` CSS (it lives in the kit). |
| 8 | Drop zone sub text does **not** list accepted formats. Keep it to the purpose line ("…available to query in chats"). Do not append `(.csv, .xls, .xlsx)`. |
| 9 | Source filter is three kit `.chip` filters in a `.dsf-filter` wrapper: **All / Artifacts / Uploaded**. Active chip carries `.is-active` + `aria-pressed="true"`. Mirrors the Metrics toolbar chip pattern — do not reimplement chip visuals. |
| 10 | Each file row shows a **source status badge** (`.badge.badge-sm`): Artifact = `badge-primary`, Uploaded = `badge-secondary` — stock kit badges, no page-local styling. Status is label-driven (not colour-only). In **Cards**: `name · date` — the modified date sits directly **next to the filename** (`.dsf-card-date`, muted, small); **file size is not shown** in Cards. The status badge (`.dsf-card-status`) is the sole element anchored to the row's right edge via `margin-left:auto`. Name flows naturally + ellipsizes when long; date and badge never clip. In **Table**, Size/Modified/Type (renamed from Source) remain separate columns. |
| 11 | Changing the source filter resets the bulk-selection context (clears `editMode` + all `selected`) so a hidden selected file from a prior filter can't be silently bulk-actioned. |
| 12 | List controls are **flat** (no filled band), consistent with the Chats Library page: a plain `.meta-row` (count + Select all / bulk actions) on top, then the `.dsf-filter` chip row below it, then the list. Do **not** reintroduce a `--card2` control band or any other new toolbar pattern without asking. |
| 13 | **Table is the only layout** — the earlier Cards/Table `.segctrl` demo toggle was removed (decided: table-only for Files). `dsfRender()` always calls `dsfRenderTable()` (kit `table.tbl`, columns: checkbox · Name · Type · Size · Modified (sortable) · kebab). |
| 14 | The list is **paginated at `PAGE_SIZE = 20`**: only the first 20 of the filtered set render, and a **Load more** button (`#dsf-more`, kit `.btn.btn-secondary.btn-sm`, showing the remaining count) reveals the next batch. Pagination resets to 20 on filter change. Select-all / count still act on the full filtered set, not just the loaded page. |
| 15 | Select-all uses the table's **shared header checkbox** (first column) — all / indeterminate (dash) / none — driving bulk selection on the full filtered set; the meta row never carries a redundant "Select all" link. |
| 16 | Order inside the library block: **filter chips → actions/meta row → list**. The actions (bulk-action bar / cards "Select all") sit **between the chips and the list**, not above the chips. |
| 17 | **File count is shown** as a persistent "N files" at rest (no jump); selecting swaps it to "N selected" + Download/Delete. Count reflects the active filter. |
| 18 | Hover tooltips must **not linger after a click**: the tooltip engine hides on `mousedown` (a clicked button often re-renders/removes itself while hovered, so no `mouseout` fires). |
| 19 | **Bulk actions render in the same meta row as the file count** (`#dsf-meta`, count/`N selected`/Download/Delete/✕ — same markup in Cards and Table, mirrors the Chats Library meta row), never a second bar. While a selection is active (`editMode`), that row gets `.dsf-meta-sticky` (`position:sticky;top:40px`, `background:var(--bg)` — the page bg, not `--card`, so scrolling rows can't show through; **no border**), so it stays reachable while the list scrolls underneath — same treatment as `chats-landing.html`'s meta row (`.cl-meta-sticky`). |
| 20 | Badge-vs-selected-row collision is fixed **in the kit, not on this page**: `kit-theme.css` gives any `.badge` a hairline (`color-mix(--ink 15%)`) **only when its `.chat-row`/`table.tbl` row is hovered/selected** (where the row bg equals the badge fill: `--state-hover`/`--tbl-row-pressed` = card2 = secondary; `--state-pressed` = brand-50 = primary). Badges are stock at rest. Do NOT reintroduce a page-local `.dsf-status` border. |
| 21 | **Clicking a file row opens a preview** in a right-hand **split panel** (`.dsf-preview`, a 3rd `.cl-shell` grid column via `.has-preview`, so the list narrows — matches prod). Panel = header (filename · Download · Close ✕) + body (document icon · filename · `size · source · date` · **Download button** — `.btn.btn-secondary.btn-sm`, same no-inline-preview treatment as the chat file panel's `.cp-fp-empty` state). Works in both Cards and Table. Opens on row-body click only (not the checkbox / kebab / menu); closes via ✕ or Escape. Previewed row's name shows in `--brand-primary`. At ≤1023px the panel overlays full-width instead of splitting. `.cl-main` carries `min-width:0` so the list column can shrink when the panel opens (otherwise the grid overflows the viewport). |
| 22 | The preview panel is **resizable** (prod-like): default width **28rem** (~448px), drag the left-edge grip (`.dsf-preview-grip`) to resize, clamped 320–720px. Implemented via the panel's own `width` (3rd grid track is `auto`) so it doesn't fight the grid `transition`; width clears on close. |
| 23 | **Tooltips always delay ~300ms on hover** — the tooltip engine's warm-up "instant re-show" was removed (`TIP_DELAY=300` on every hover). Every `data-tip` (sidebar, kebab-less icons, meta ✕, sticky meta row, preview Download/Close) routes through this one engine. |
| 24 | Every file row's kebab menu carries **Add to Chat · Rename · Download · Delete** (in that order; Delete is `.danger`) — `.menu.chat-row-menu`. **Add to Chat navigates to `chat-landing.html?attach=<filename>`**, which pre-fills the composer's file chip (`clPickFile()`) on load — the same one-file-attachment affordance the composer already has. |
| 25 | **Modified column is sortable.** Clicking the "Modified" header (`.dsft-sortbtn`, `data-action="sort-date"`) toggles `sortDir` between `desc` (newest first, default) and `asc` (oldest first); the header carries `aria-sort` and a chevron that flips to match. Sort applies to the full filtered set before pagination, using each file's seed order (`sortIdx`, 0 = newest) as the recency proxy — the seed's relative-date strings ("3d ago", "2 months ago", …) aren't machine-sortable. New uploads take decreasing negative `sortIdx` values so "just now" rows sort above the seed under newest-first. |
| 26 | **First-run empty state** (0 files) is the kit `.empty-state` (`#dsf-empty`, outside `.dsf-library`) with the shared **`.es-illu` fading-cards illustration** (mirrors the file list it replaces — same concept prod ships on `/files`; user-picked over the 48px glyph 2026-07-21), title "No files yet", helper "Files you upload or create in chats will appear here" — **no trailing period**, **no CTA** (the always-visible drop zone above is the page's upload CTA). |
| 27 | **Filter no-match empty state** — when files exist but the active source filter yields 0 (e.g. every Uploaded file was deleted while that chip is active), the list region renders the kit `.empty-state` with the shared `.es-illu` fading-cards illustration, "No matches found", and a switch-back-to-All hint. Filter chips stay visible; the meta row hides (`0 files · sort` is noise). |
| 28 | **Loading skeleton** — same `.sk-loaded` / `.sk-skel` + `html.skel-on` pattern as the Connections page, driven by the topbar **Loaded/Skeleton** SegmentedControl (a demo control, never wired to `.prod`). The skeleton mirrors the library block: 3 filter-chip ghosts + count line + the 6-column `table.tbl.dsf-tbl` with real header labels and 5 ghost rows. The **drop zone stays real** during loading — it is static page chrome, not loaded data. The topbar also carries a **Filled/Empty** segctrl (demo control) that swaps the seed out/in to preview the first-run empty state. |
| 29 | **Rename opens a dialog** (`#dsf-ren-dlg`) — kit `.dlg-overlay`/`.dlg` shell, same recipe as the Connections "Edit connection" dialog (mirrors prod's Rename Chat modal): title "Rename file" (**popup titles are Sentence case** — user directive 2026-07-21) + ✕, one-line helper ("Give your file a recognizable name", no trailing period), a `.field` input **prefilled with the current name with the basename preselected** (extension stays out of the selection), footer `Cancel` (secondary) + `Save` (primary). Saves on Enter, closes on Cancel / ✕ / Escape / backdrop click; empty input saves nothing. |
| 30 | **Delete always confirms** (`#dsf-del-dlg`) — both the row kebab's Delete and the bulk Delete open a kit `.dlg` confirm (same recipe as Connections "Disconnect?" / prod's "Delete chat?"): title "Delete file?" / "Delete N files?" (Sentence case, dynamic count), body "This action can't be undone. The (selected) file(s) will be permanently removed.", footer `Cancel` (secondary) + `Delete` (`.btn-destructive`, focused on open). Never delete straight from the menu/meta row without this confirm. Escape priority on the page: delete/rename dialog → open kebab menu (file rows **and** sidebar chats) → preview panel. |
| 31 | **Sidebar chat menus are functional** — only ONE `.sbx-chat` menu may be open at a time (opening a kebab closes the rest; outside click / Escape dismisses — a lingering open menu from the row above otherwise floats over the next section and reads as the wrong row's menu). Menu actions work: **Rename** opens the shared rename dialog re-worded for chats ("Rename chat" / "Give your chat a special name", full name preselected); **Delete** opens the shared confirm re-worded ("Delete chat?" / "…The chat will be permanently removed."); **Pin / Unpin** moves the row between the Pinned and Recent sections and flips its own menu label. The dialogs re-word per flow — the file flows always reset their own wording on open. |

## Layout / composition changes

- **Drop zone — dashed drop target.** Centered dashed zone (reads as drag-and-drop): upload icon inline with the title (`.dsf-drop-head`), helper line + Browse button below, `1.25rem 1.5rem` padding. Sub line has no accepted-formats parenthetical and uses `text-wrap:balance`.
- **Source filter.** Kit `.chip` filters (All / Artifacts / Uploaded) in a flat `.dsf-filter` row. Filtering re-derives the list, count, and selection from the visible set (`dsfVisible()`); changing the filter resets the selection context.
- **Per-row source status.** `FILES[].origin` (`'artifact' | 'uploaded'`) drives a `.badge.badge-sm` — Artifact = `badge-primary`, Uploaded = `badge-secondary`. New uploads default to `origin:'uploaded'`.
- **Flat controls (Chats-Library-consistent).** `.ds-page-body` is a flat flex column; the library block is a plain `.meta-row` (count + Select all) + `.dsf-filter` chip row + list — **no filled control band** (the earlier `--card2` `.dsf-toolbar` band was a new pattern and was removed).
- **Table only.** The Cards/Table `.segctrl` demo toggle was removed; `dsfRender()` always calls `dsfRenderTable()` (kit `table.tbl`). Shared drop zone, filters, selection, and delete logic are unaffected. (Cards markup/CSS `.dsf-list.is-cards`, `.dsf-card-date`, `.dsf-card-status` removed — cascading delete.)
- **Pagination.** Seed grew to 24 rows; the list renders `PAGE_SIZE = 20` at a time with a **Load more** button (`#dsf-more`) that reveals the next batch and shows the remaining count. Resets to 20 on filter change; count + Select all operate on the whole filtered set. Works in both layouts.
- **File preview (as prod).** Clicking a file row opens a right-hand split panel (`.dsf-preview`) — the list narrows via a 3rd `.cl-shell` grid column (`.has-preview`), matching prod's `/sources` Files behaviour. Panel shows filename header + Download + Close, and a body with a document icon, filename, and `size · source · date`. Row-body click only; ✕/Escape close; previewed row name highlights in brand. Overlays full-width at ≤1023px. `previewId` drives `dsfRenderPreview()`.
- **Sticky selection bar.** The meta row (count / `N selected` / Download / Delete / ✕) pins `position:sticky;top:40px` below the topbar whenever a selection is active, so it stays visible while the list scrolls beneath it — painted with the page `--bg`, no border. Same behaviour as the Chats Library meta row.
- **Kebab menu — Add to Chat.** Every file row's `···` menu now opens with **Add to Chat · Download · Delete**. Add to Chat navigates to `chat-landing.html?attach=<filename>`, which arrives with the file already attached in the composer.
- **Sortable Modified column.** Clicking the "Modified" header toggles newest-first/oldest-first ordering across the whole filtered set (`sortDir`), with an `aria-sort` header and a flipping chevron.
- **First-run empty state — fixed vs prod.** Prod's `/files` empty state shows a fading-rows illustration, an oversized bold title and a helper with a trailing period ("Files will appear here as they are uploaded or opened."). Expected keeps the illustration concept but rebuilds it from the kit `.empty-state`: shared `.es-illu` fading-cards SVG, `.empty-title` "No files yet", `.empty-msg` "Files you upload or create in chats will appear here" (no trailing period, `text-wrap:balance`). No CTA — the drop zone above is the upload CTA.
- **Filter no-match empty state.** New — prod has none. Kit `.empty-state` + shared `.es-illu`, "No matches found", switch-to-All hint; chips stay, meta row hides.
- **Loading skeleton.** New — `.sk-loaded`/`.sk-skel` regions flipped by `html.skel-on` (topbar Loaded/Skeleton segctrl): 3 chip ghosts + count ghost + 5-row table skeleton mirroring the real 6 columns; drop zone stays real. A topbar Filled/Empty segctrl previews the first-run empty state.

## Out of scope

- Reuses existing `.chip` (filter), `.badge` (status), `table.tbl` (table view) as-is. **DropZone (`.dsf-drop*`) was promoted to a kit component** — CSS moved to `kit-theme.css`, storybook `#dropzone` added, `changes/DropZone.md` created. Remaining page-only glue: `.dsf-library`, `.dsf-filter`, `.dsf-tbl*`.
